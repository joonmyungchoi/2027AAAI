# Sanity 합성 결과 보고 — 10 instance × 4 persona

> 작성: 2026-06-11. 스크립트: `synth_sanity.py`, 전체 출력: `sanity_output.md`.
> 목적: gt_design_v1 규칙이 기계 합성 가능한지 + 수치 초기값 검증.

---

## 1. 결과 요약

- **10 instance × 4 persona = 40 시퀀스 합성 성공**, 미발사 경고 0.
- **변별력**: 약속 있는 8/8 instance에서 4 persona 모두 서로 다른 GT 시퀀스. no-op 2개는 전 persona 동일(전부 silent) — 의도대로.
- **수치 확정**: B_early=30 / B_late=10 / prep=10 — 미리·임박 첫 fire가 항상 다른 그리드에 떨어짐 (예: KTX에서 15:15 vs 15:45). **확정**.
- 비 보정: 도보 구간 ≥15분이면 +10(승격 후보), 미만 +5(미세) — 경계 양측 시험 동작 확인.

## 2. 발견 결함 → 규칙 수정 (gt_design v1.1 반영)

| # | 결함 (sanity가 잡음) | 수정 규칙 |
|---|---|---|
| R1 | 동적 이벤트가 발사창을 그리드 사이로 후퇴시키면 첫 알림이 영영 안 나감 (I02·임박) — 빈번도 필터가 첫 알림에도 적용된 탓 | **빈번도 필터는 재알림에만 적용.** 첫 알림 미발사 상태의 이벤트 trigger는 band 시작 도달 시 즉시 fire |
| R2 | 그리드가 발사창을 건너뛰는 엣지 | **첫 알림 = band 시작(ft−7.5) 이후 첫 trigger에서 무조건 fire** (band 포함 일반화) |
| R3 | 이동 판정이 재계산된 출발시각 기준이라 안내와 불일치 | **이동 판정 = 마지막 안내한 출발시각 기준** (user는 안내를 따른다고 가정) |
| R4 | 상대 +60 지연 시 약속이 18:00로 밀려도 그리드가 17:00에 끝남 (I07) | **관찰 창은 T_appt 갱신을 따라 연장** — env 루프는 현재 유효 약속시각까지 tick |
| R5 | 이동 중 변경공지가 과거 출발시각을 안내 ("새 출발 15:35") | **이동 중 content = 도착 영향** ("도착예정 17:15 vs 약속 17:10"), 출발시각 안내는 출발 전만 |
| R6 | 필요 출발시각이 이미 지난 강제 이벤트 (I04 위험기상) → 과거 시각 안내 | **depart < now → "즉시 출발 + 지각 N분" content** (규칙만 확정, 프로토타입 미구현) |

## 3. 윈도 정책 확정

- 기본 90분. 단 `travel + buffer + prep + 7.5 > 90` 이면 그리드 단위 확장 (KTX×미리 = 120분, 그리드 9개).
- R4와 결합 — 최종 그리드 수는 instance·persona에 따라 7~13개로 가변. 결정 포인트 수가 늘어나는 쪽이라 채점 pool엔 이득.

## 4. 생성기 제약 추가 (template_matrix 갱신 필요)

1. **E4(state) 겹침 기준 재정의** — persona마다 발사창이 달라 "발사창과 겹침"이 동시 성립 불가. → 기준을 **강제/승격 이벤트 fire 시각(persona 무관)**으로 변경. I01의 통화(16:20~40)가 16:35 승격 fire를 덮은 패턴이 모범.
2. **미세 이벤트 일부는 출발 전 배치** — I01처럼 미세 이벤트가 전부 이동 중이면 빈번도 축이 decision 레벨에서 변별 안 됨 (modality/content로만 갈림). 미세 이벤트(E1 비, E3 +5)의 ≥1개는 **가장 늦은 persona 출발 전**에 오도록 제약.
3. **지각 불가피 셀** — 강제 +Δ가 출발 여유를 초과하는 조합은 의도된 셀("즉시 출발" 시험)로만 허용, 우연 발생은 가드.
